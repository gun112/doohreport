## 적용 순서

### 1. components 폴더 생성
```bash
mkdir -p components
```

### 2. 파일 복사
다운로드 받은 파일들을 위 경로대로 복사

### 3. 기존 /dashboard 경로 처리
기존 `/dashboard/...` 파일들은 일단 그대로 둬도 됨.
새로운 `/map`, `/report`, `/recommend`, `/simulate`가 우선됨.

### 4. 로컬 테스트
```bash
npm run dev
```
→ `localhost:3000` 접속 → 랜딩 페이지 확인
→ `localhost:3000/map` → 지도 페이지 확인

### 5. 배포
```bash
git add .
git commit -m "랜딩 페이지 v2 + 지도 UI"
git push
```
→ Vercel 자동 배포

## 주의사항

1. **tsconfig.json** — `@/components` 경로가 동작하려면 `paths` 설정 필요:
```json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./*"]
    }
  }
}
```
(Next.js 기본 생성 시 이미 있을 가능성 높음)

2. **globals.css** — 기존 globals.css에 Tailwind directives가 있다면 그대로 유지해도 됨.
새 컴포넌트들은 전부 인라인 스타일이라 충돌 없음.

3. **기존 API 라우트** — 전혀 건드리지 않음. 그대로 동작.

4. **Supabase** — 아직 연결 안 됨. 리포트 페이지에서 Supabase 연동은 다음 스텝.
